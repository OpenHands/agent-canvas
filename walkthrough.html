Issue #348 This walkthrough covers the concrete changes in this PR.

1 What Changed

✗ Before

  • npm run dev → runs Docker mode by default
  • Fails if Docker isn’t installed
  • Step 1: “Check Backend” — just verifies connection, single backend
  • No way to run Local + Docker simultaneously

✓ After

  • npm run dev → always works (local mode, no Docker needed)
  • Step 1: “Choose Backend” — multi-select Local + Docker
  • Mini setup server bridges frontend ↔ Docker lifecycle
  • Docker is started on-demand through the UI

2 Architecture

npm run dev ├─ Vite dev server :3001 ├─ Local agent-server :18000 (always started) ├─ Automation backend :18001 ├─ Ingress proxy :8000 (routes all traffic) ├─ Mini Setup Server :18099 ← NEW │ ├─ GET /setup/status → { dockerInstalled, dockerRunning, dockerBackendRunning } │ ├─ POST /setup/docker → Start Docker backend, returns { host, port } │ └─ DELETE /setup/docker → Stop Docker backend │ └─ [spawned on demand by setup server] Docker agent-server :18002 The ingress proxy routes /setup/* to the setup server, so the browser frontend accesses it via the same origin.

Why a setup server?

  • The browser cannot spawn Docker containers
  • The Node.js npm run dev process can
  • A tiny HTTP server (~110 lines) bridges the gap
  • 127.0.0.1 only — not network-exposed
  • Reuses Docker logic from existing dev-docker.mjs

5 What happens when the user clicks “Next”

If only Local is selected

Nothing to do — local backend is already running since npm run dev. Proceeds immediately.

If Docker is selected

  1. User enters workspace directory and clicks “Start Docker Backend”
  2. Frontend calls GET /setup/status to check Docker availability
  3. If Docker is available, calls POST /setup/docker with { projectPath }
  4. Setup server spawns Docker container on port 18002
  5. Shows “Docker backend started” confirmation
  6. “Next” button becomes enabled once Docker is running or Local is selected

6 Setup Server API

EndpointMethodResponse
/setup/status GET { dockerInstalled, dockerRunning, dockerBackendRunning, dockerBackendPort }
/setup/docker POST Body: { projectPath } → { status, host, port, url }
/setup/docker DELETE { status: "stopped" }

All endpoints return JSON with CORS headers. The ingress proxy routes /setup/* to the setup server, so the frontend uses the same origin.
